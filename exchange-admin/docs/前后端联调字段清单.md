# 《前后端联调字段清单》

> **适用范围**：Vue3 前端 + Java 后端 + MySQL 5.7 的加密货币交易所项目。  
> **基线说明**：HTTP 路径与字段命名以下列前端实现为联调基准（`src/api/**`、类型定义）；后端若存在差异，请在联调单中标注「后端实际字段 / 映射方案」。  
> **Base URL**：`VITE_API_BASE`（默认 `/api`），即浏览器请求形如：`/api/v1/...`。

---

## 0. 全局约定

### 0.1 统一响应包体 `ApiResult<T>`

除特别说明外，HTTP JSON 根结构建议为：

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | `boolean` | 业务是否成功 |
| `code` | `string` | 业务码（与前端 `ErrorCode` 对齐，如 `AUTH_TOKEN_EXPIRED`） |
| `message` | `string` | 人类可读说明 |
| `data` | `T` | 业务载荷；**前端 `apiGet`/`apiPost` 取的是 `data` 内层** |
| `traceId` | `string` | 可选，链路追踪 |
| `timestamp` | `number` | 可选，服务端时间戳 |

**前端引用**：`src/types/api.ts` → `ApiResult`；`src/api/common/http.ts` 拦截器对 `success === false` 与部分 `code` 做统一提示/登出。

### 0.2 鉴权

| 项 | 说明 |
|----|------|
| Header | `Authorization: Bearer <accessToken>` |
| 前端写入 | `src/api/common/http.ts` 请求拦截器从 `tokenStorage` 读取 |

### 0.3 前端常用错误码（需后端对齐）

| `code` | 含义（前端处理） |
|--------|------------------|
| `AUTH_TOKEN_MISSING` | 未带 Token |
| `AUTH_TOKEN_EXPIRED` | Access Token 过期 |
| `AUTH_TOKEN_INVALID` | Token 无效 |
| `AUTH_REFRESH_INVALID` | Refresh 无效（若启用刷新） |

**前端引用**：`src/enums/errorCode.ts`。

## 1. 登录认证

### 1.1 用户登录

| 项 | 内容 |
|----|------|
| **接口名称** | 用户登录 |
| **路径** | `POST /v1/auth/login` |
| **请求方式** | `POST` |
| **鉴权** | 否 |

**请求参数（Body JSON）**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `principal` | `string` | 是 | 登录名：当前前端用作**邮箱** |
| `password` | `string` | 是 | 密码 |

**返回字段（`data` 内层，与 `LoginResult` 对齐）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `accessToken` | `string` | 访问令牌，后续 API 放 Header |
| `expiresIn` | `number` | 过期秒数（前端可做本地过期提示） |
| `refreshToken` | `string` | 可选，刷新令牌 |
| `user` | `object` | 用户信息，见下表 |

**`user`（`UserProfile`）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `userCode` | `string` | 用户唯一展示 ID |
| `nickname` | `string` | 可选 |
| `emailMasked` | `string` | 可选，脱敏邮箱 |
| `avatarUrl` | `string` | 可选 |
| `kycLevel` | `number` | 可选 |
| `status` | `number` | 可选，账号状态 |
| `roles` | `string[]` | 可选，角色（如后台权限） |

**字段说明**：登录成功后前端将 `accessToken` 持久化，并保存 `user` 至 Pinia；路由守卫与资产/用户中心等依赖登录态。

**前端页面 / 模块**：`LoginPage.vue`；`stores/auth.ts` → `loginApi`（`src/api/auth/index.ts`）。

### 2.1 全市场 ticker 列表

| 项 | 内容 |
|----|------|
| **接口名称** | 全市场行情列表 |

| **请求方式** | `GET` |

**请求参数（Query）**

| 字段 | 类型 | 必填 | 说明 |

### 4.2 现货下单 / 撤单
